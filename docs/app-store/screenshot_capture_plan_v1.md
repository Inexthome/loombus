# Loombus Screenshot Capture Plan v1

## Goal

Prepare first-pass App Store and Google Play screenshots that show Loombus as a mobile-first high-signal discussion platform.

## Recommended screenshot set

Capture these 8 phone screenshots first:

1. Home / Signal Hub  
Show the mobile Home tab with "What matters now," Create a discussion, Discussions, Alerts, Following, and Saved.

2. Discussions feed  
Show the signal-card feed with Signal Score and Replies / Saves / Views.

3. Discussion detail reader  
Show a discussion detail page with the reader shortcuts: Reply, Replies, AI tools.

4. Create composer  
Show the mobile composer with title, topic, body, and Publish Discussion.

5. Alerts inbox  
Show the compact notifications inbox and filter row.

6. People discovery  
Show mobile-native member cards and Follow / Following controls.

7. Following Signal feed  
Show followed-discussion signal cards.

8. Menu overlay  
Show the app-style Menu overlay with Primary / Discovery / Account groups.

## Apple screenshot notes

Apple accepts 1 to 10 screenshots in .jpeg, .jpg, or .png.

Recommended first iPhone capture target:
- 6.9-inch iPhone portrait screenshots where possible.

Useful accepted portrait dimensions include:
- 1260 x 2736
- 1290 x 2796
- 1320 x 2868

If the app supports iPad, prepare iPad screenshots too.

Useful accepted 13-inch iPad portrait dimensions include:
- 2064 x 2752
- 2048 x 2732

## Google Play screenshot notes

Google Play requires at least 2 screenshots across device types and allows up to 8 screenshots per supported device type.

Recommended first Android capture target:
- 8 phone screenshots
- Portrait orientation
- At least 1080 px wide
- 9:16 aspect ratio where possible
- JPEG or 24-bit PNG with no alpha

## Capture guidance

Use real app screens only.

Avoid:
- "Best," "#1," "Top," rankings, awards, or download claims
- Sale/discount language
- Screenshots that hide the actual interface
- Blurry, stretched, or compressed images
- Overly heavy marketing text

Recommended approach:
- Capture clean in-app screens first.
- Add marketing frames/captions later only if needed.
- Keep the first three screenshots focused on actual core UI:
  1. Home / Signal Hub
  2. Discussions feed
  3. Discussion detail reader

## Manual capture checklist

Before taking screenshots:
- Use a clean account state if possible.
- Make sure sample discussions look useful and safe.
- Avoid exposing private email, private notes, admin-only controls, or test data.
- Prefer content that represents Loombus clearly:
  - thoughtful titles
  - topic tags
  - visible Signal Score
  - clean reply examples
  - no controversial placeholder content
